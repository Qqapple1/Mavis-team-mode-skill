🧠 Mavis Team Mode

A community port of Mavis's Leader + Workers + Verifier workflow, packaged as a portable Agent Skill for Zcode 3.x. Runs on any model your Zcode is configured for.

Quick install

# macOS / Linux / Git Bash / WSL
bash <(curl -fsSL https://raw.githubusercontent.com/Qqapple1/Mavis-team-mode-skill/main/scripts/install.sh)

# Or git clone + manual
git clone https://github.com/Qqapple1/Mavis-team-mode-skill.git ~/mavis-team-mode-skill
bash ~/mavis-team-mode-skill/scripts/install.sh

# Windows PowerShell
git clone https://github.com/Qqapple1/Mavis-team-mode-skill.git $env:USERPROFILE\mavis-team-mode-skill
cd $env:USERPROFILE\mavis-team-mode-skill
powershell -ExecutionPolicy Bypass -File scripts\install.ps1

# Verify
bash scripts/install.sh --doctor    # or: powershell -File scripts\install.ps1 -Doctor

The workflow

┌─────────────────────────────────────────────────┐ │ User: "用 Mavis team mode 帮我做 X" │ └─────────────────────┬───────────────────────────┘ ▼ Phase 1: SCOPE + PLAN (Leader, ~2 min) - Decompose into 1..N subtasks - Decide parallel vs serial - Pick sub-agent type per subtask │ ▼ Phase 2: DISPATCH (parallel where possible) ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │ Worker │ │ Worker │ │ Worker │ │ Worker │ │ Coder │ │ Tester │ │Research│ │ Doc │ └────┬───┘ └────┬───┘ └────┬───┘ └────┬───┘ ▼ ▼ ▼ ▼ Phase 3: INTEGRATE (Leader synthesizes) ▼ Phase 4: VERIFY (spawn Verifier, separate context) ▼ ┌────────┐ ┌────────┐ │ PASS │ │ FAIL │ └────┬───┘ └────┬───┘ │ ▼ │ Phase 5: ITERATE (max 3x, back to P2) ▼ Phase 6: DELIVER (summary + path to artifacts)

Features

Parallel Workers

Decompose any task into 1..N sub-tasks, dispatch concurrently via Zcode's sub-agent system.

Independent Verifier

A separate sub-agent (different conversation) reviews the result. Same model as Leader = possible bias, see docs.

Iterate with guardrails

Up to 3 retry cycles before delivering with explicit caveats. Bounded cost.

Progressive disclosure

Only SKILL.md loads on trigger. Agents / examples / references load on demand.

7 sub-agent roles

leader, verifier, worker-coder, worker-tester, worker-researcher, worker-doc-writer, worker-reviewer.

4 worked examples

Refactor large module, bug hunt, new feature, research-then-implement. Each is copy-pasteable.

Defense-in-depth prototype

Real working Todo app with CORS whitelist, body cap, threading lock, 48 e2e tests (20 + 23 + 5).

Multi-platform

bash + PowerShell installers. Tests on Ubuntu + macOS + Windows, Python 3.8 → 3.12.

Why this exists

Mavis's Team Mode (Leader + Workers + Verifier) is a well-designed multi-agent workflow. But it ships only with the M3 model in MiniMax Code. This skill brings the same workflow to any Zcode 3.x setup — so you can run the workflow on whatever model your Zcode is configured for.
